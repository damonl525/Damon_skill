# Statistical Review Summary

作为临床统计师，系统性总结参考研究资料的标准化模板。

## Overview

This skill helps you:
- 创建标准化的临床研究总结文档
- 根据试验类型（早期/确证性）自适应调整关注重点
- 系统性整理试验设计、统计方法和结果
- 记录监管机构的关注点和态度
- 深入分析值得关注的统计方法论
- 为药物开发决策提供结构化参考

## 试验类型识别

使用模板前，先根据可用资料判断试验类型，选择合适的模板路径：

| 试验类型 | 特征 | 核心关注点 |
|----------|------|------------|
| **I/II期剂量递增** | 开放标签、非随机、小样本、剂量探索 | 递增方法、DLT定义、决策规则、RP2D |
| **II期疗效探索** | 单臂或随机、信号生成、生物标志物 | 疗效信号、终点选择、Go/No-Go标准 |
| **III期确证性** | 随机对照、大样本、确证性设计 | 多重性控制、中期分析、确证性证据 |

> **注意**：模板中的「条件性章节」根据试验类型决定是否填写。不适用的章节标注原因后跳过，不要强行填写。

## 使用场景

### 1. 文献回顾总结
- 总结已发表的临床试验论文
- 整理多个研究的关键发现
- 比较不同研究的统计方法

### 2. 监管文件准备
- 准备IND/NDA申报材料中的统计部分
- 总结FDA审评报告的关键要点
- 整理咨询委员会的讨论要点

### 3. 内部决策支持
- 为项目团队提供研究概览
- 支持药物开发策略讨论
- 评估统计方法的适当性

## 模板结构

### 核心章节

| 章节 | 适用范围 | 内容要点 |
|------|----------|----------|
| **1. 上市依据** | 有监管文档时 | 核心试验、支持性试验、药物/适应症 |
| **2. 试验设计** | 所有试验 | 设计类型、样本量、随机化、治疗、盲法 |
| **3. 主要终点** | 所有试验 | 终点定义、类型适配引导、临床意义 |
| **4. 统计分析方法** | 所有试验 | 分析人群、主要分析、多重性、敏感性 |
| **5. 主要结果** | 所有试验 | 关键结果数据、亚组分析 |

### 条件性章节

| 章节 | 适用范围 | 内容要点 |
|------|----------|----------|
| **6. 剂量递增设计** | I/II期 | 递增方法、DLT定义、决策规则、RP2D |
| **7. 中期分析与删失规则** | II/III期 | 中期分析计划、alpha消耗、删失规则 |
| **8. 监管看法** | 有监管文档时 | 咨询委员会、审评关注点、最终结论 |

### 开放式章节

| 章节 | 适用范围 | 内容要点 |
|------|----------|----------|
| **9. 值得关注的统计方法** | 所有试验 | **不预设分类**，根据试验特点自行组织 |

## 使用方法

### 基本流程

1. **识别试验类型**
   - 根据资料确定试验阶段和设计类型
   - 选择需要填写的条件性章节

2. **准备研究资料**
   - 临床试验方案（Protocol）——关注设计、随机化、终点定义
   - 统计分析计划（SAP）——关注分析方法、多重性、敏感性分析
   - 临床研究报告（CSR）——关注结果
   - 发表论文（Publication）——关注最终报告的数据
   - FDA审评报告（StatR, MedR）——关注监管视角
   - 咨询委员会记录——关注争议点

3. **按模板填写**
   - 保持客观、准确的描述
   - 关键数字标注来源（文档名称+页码/章节号）
   - 不确定的信息明确标注，不要猜测

4. **质量审核**
   - 数值一致性检查（同一数字在不同章节是否一致）
   - 可追溯性检查（关键结论能否找到原文出处）
   - 方法一致性检查（描述的方法是否与实际使用的方法匹配）

### 多文档冲突处理

当不同文档间信息不一致时：
1. **SAP > Protocol**：统计方法以 SAP 为准（SAP 通常更晚定稿）
2. **CSR > SAP > Protocol**：结果以 CSR 为准
3. **Publication > CSR**：发表数据为最终数据（可能包含更新）
4. **明确标注差异**：记录不一致之处，注明各文档的说法
5. **不要擅自裁定**：如果无法确定哪个正确，标注「待确认」

## 最佳实践

### 1. 信息提取原则

**溯源优先**
- 每个关键数字标注来源文档和页码
- 区分「原文明确说明」vs「推断得出」vs「信息缺失」
- 多个文档交叉验证关键信息
- **试验注册号（NCT/EudraCT）、protocol ID 等唯一标识符不可盲信资料脚注**：会议 poster/abstract/幻灯片常有誊写错误，必须回查 ClinicalTrials.gov / EudraCT 原始记录核实（标题、申办方、期别、人群要对得上）。实例：KRYSTAL-10（ESMO GI 2026）poster 脚注 NCT 号印成 NCT02872116（全 KRYSTAL 系列查无此号），正确注册号为 NCT04793958。

**方法优先于结果**
- 先理解「怎么分析的」，再报告「结果是什么」
- 统计方法的选择直接影响结果的解读
- 不理解方法就报告结果，不如不报告

### 2. 统计方法记录

**记录层次**
- 第一层：使用了什么方法（名称）
- 第二层：为什么选择这个方法（适用条件）
- 第三层：关键参数和假设（具体设置）
- 第四层：结果的稳健性（敏感性分析是否支持）

**值得关注的判定标准**
- 方法论上的创新或不常见的选择
- 对监管决策有影响的方法
- 假设可能不满足时的处理策略
- 与同类试验不同的方法选择
- 复杂统计方法需要专门解释（如区间删失模型、图形多重性控制等）

### 3. 结果报告

**必备要素**
- 效应量和置信区间（不只是 p 值）
- 分析人群（FAS/PPS/SS）
- 与主要分析方法的对应关系

**避免**
- 只报告 p 值不报告效应量
- 混淆不同分析人群的结果
- 将探索性分析结果当作确证性证据

## 输出格式

### Markdown 格式规范（重要）

输出为标准 Markdown，**使用流式排版（自由格式文本），不要用代码块（```）包裹内容**。

> **为什么**：代码块在 Markdown 阅读器中渲染为等宽 `<pre>` 块，**长行不自动换行**，会产生横向滚动条，内容显示不全、阅读体验差。流式 Markdown（段落、列表、表格）会自动换行适配窗口宽度。

**排版规则：**

| 元素 | 用法 |
|------|------|
| 段落/说明 | 普通文本段落，自然换行 |
| 列举 | 无序列表 `- ` / 有序列表 `1.` |
| 表格型数据（分组对照、参数、结果） | **优先用 Markdown 表格** `| ... |`，而非挤在文字里 |
| 强调 | `**加粗**`；行内术语/代码可用 `` ` `` 行内代码（仅单个词，不要围起整段） |
| 分章节 | `---` 水平线**仅用于章节分隔**，不要用它或代码块围起内容 |
| 注释/引用/原文摘录 | `>` 引用块 |
| 公式 | 行内 `$...$` 或独立 `$$...$$`，不用代码块 |

**不要这样做：**

```text
❌ 把整段内容放进三反引号代码块：
### 5.1 主要终点
​```
PFS：治疗组 5.6 个月，对照组 4.5 个月；HR 0.66 ...
​```
```

**应当这样做：**

- 治疗组中位 PFS 5.6 个月，对照组 4.5 个月；**HR 0.66**（关键数据可用表格呈现）。

> 唯一允许使用代码块的场景：展示**真正的代码/命令**，或必须保留对齐的 ASCII 图（如决策树），且应尽量简短。

## 质量检查清单

### 完整性检查
- [ ] 试验类型是否正确识别？
- [ ] 各核心章节是否填写？
- [ ] 条件性章节是否根据试验类型选择？
- [ ] 不适用的章节是否标注了原因？

### 准确性检查
- [ ] 关键数字是否有来源标注（文档+页码）？
- [ ] 同一数字在不同章节中是否一致？（如样本量在2.2和4.x中）
- [ ] 统计方法描述是否与实际使用的方法匹配？
- [ ] p值、置信区间等是否与原文一致？
- [ ] 试验注册号（NCT/EudraCT）是否已回查 ClinicalTrials.gov / EudraCT 核实？（会议 poster 脚注常有誊写错误，不可盲信）

### 深度检查
- [ ] 统计方法是否记录了选择理由？
- [ ] 敏感性分析是否被讨论？
- [ ] 值得关注的方法是否解释了「为什么值得关注」？
- [ ] 不确定或缺失的信息是否被标注？

### 批判性检查
- [ ] 是否识别了方法论的局限性？
- [ ] 假设是否可能不满足？（如 PH 假设、比例优势假设等）
- [ ] 是否讨论了结果对方法选择的敏感性？
