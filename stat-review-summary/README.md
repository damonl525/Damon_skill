# Statistical Review Summary Skill

## 简介

这是一个专为临床统计师设计的技能，用于系统性总结参考研究资料。它提供了一个标准化的模板，帮助您快速、准确地整理临床试验的关键信息，包括试验设计、统计方法、结果和监管观点。

## 主要功能

- ✅ 标准化的总结模板
- ✅ 详细的填写指南
- ✅ 完整的示例参考
- ✅ 质量检查清单
- ✅ 支持中英文文档

## 快速开始

### 1. 准备研究资料

收集以下文档：
- 临床试验方案（Protocol）
- 统计分析计划（SAP）
- 临床研究报告（CSR）
- FDA审评报告（StatR, MedR）
- 咨询委员会记录

### 2. 使用模板

1. 复制 `template.md` 文件
2. 按照模板结构填写内容
3. 参考 `example.md` 了解填写方式

### 3. 审核和完善

- 检查数据一致性
- 确保关键信息完整
- 请同事审阅

## 模板结构详解

### 第一部分：上市依据的基础

**目的**：概述药物开发的整体策略和依据

**应包含内容**：
- 核心试验（关键性研究）
- 支持性试验（早期研究、安全性研究等）
- 药物名称和适应症
- 目标患者人群

**示例**：
```
核心试验：Study 301（一项III期、开放标签、随机、对照试验）。
支持性试验：Study 101 & 102（I期，主要用于安全性支持）。
药物/适应症：LUXTURNA 用于治疗 RPE65 基因突变相关视网膜营养不良。
```

### 第二部分：试验设计

**目的**：详细描述试验的设计特征

**应包含内容**：
- 设计类型（随机、对照、盲法等）
- 样本量和分组
- 随机化方法和分层因素
- 治疗方案
- 盲法实施

**示例**：
```
设计类型：III期、多中心（2个中心）、开放标签、随机、对照。
样本量：共 31 例患者随机化（治疗组 21 例，对照组 10 例）。
随机化：2:1 分配，分层因素：年龄、基线移动性测试通过水平。
```

### 第三部分：主要终点

**目的**：明确定义主要疗效终点

**应包含内容**：
- 终点名称和定义
- 临床意义
- 评分或测量方法
- 改善的定义

**示例**：
```
终点名称：MLMT评分变化（双眼，从基线到第 1 年）。
临床意义：评估患者在不同光照水平下通过障碍物路径的能力。
评分逻辑：得分 = 能通过的最低光照水平对应的分值。
```

### 第四部分：统计分析方法

**目的**：描述统计分析的具体方法

**应包含内容**：
- 主要分析方法
- 多重性控制策略
- 缺失数据处理
- 敏感性分析
- 样本量计算

**示例**：
```
主要分析：Wilcoxon 秩和检验（使用 exact p-value）。
置换检验：重复再随机 10,000 次，作为主要推断依据。
多重性控制：采用层级检验控制 I 类错误。
缺失数据处理：明确定义了 4 种情况。
```

### 第五部分：主要结果

**目的**：呈现关键的研究结果

**应包含内容**：
- 主要终点结果
- 效应量和置信区间
- p值
- 关键的亚组分析
- 图形展示（如有）

**示例**：
```
MLMT 评分变化 (中位数)：治疗组 +2，对照组 0，p=0.001。
改善≥2 分的患者比例：治疗组 52%，对照组 10%。
```

### 第六部分：监管看法

**目的**：记录监管机构的态度和关注点

**应包含内容**：
- 咨询委员会投票结果
- 监管机构关注的问题
- 审稿人的评论
- 最终结论

**示例**：
```
咨询委员会投票：16 : 0 一致认为"总体获益-风险概况有利"。
监管关注：序数量表的信息损失、天花板效应。
最终结论：FDA 统计审评结论为"提供了统计证据支持批准"。
```

### 第七部分：值得关注的统计方法

**目的**：突出显示特殊的统计方法或创新

**应包含内容**：
- 小样本量方法
- 贝叶斯方法
- 自适应设计
- 替代终点
- 中间指标

**示例**：
```
基于前期 Phase 1 数据，采用Wilcoxon rank sum test模拟计算样本量。
主要终点采用基于 Wilcoxon 秩和检验统计量的非参数置换检验。
```

## 使用场景

### 场景1：文献回顾

**任务**：总结一篇已发表的临床试验论文

**步骤**：
1. 阅读论文全文
2. 按照模板提取关键信息
3. 标注数据来源（页码、表格号）
4. 记录任何疑问或需要进一步确认的内容

### 场景2：监管文件准备

**任务**：准备IND/NDA申报材料的统计部分

**步骤**：
1. 收集所有相关研究资料
2. 按照模板整理信息
3. 确保与申报策略一致
4. 请团队审阅和确认

### 场景3：内部决策支持

**任务**：为项目团队提供研究概览

**步骤**：
1. 快速提取关键信息
2. 突出重要的监管观点
3. 标注需要讨论的问题
4. 准备会议材料

## 质量检查清单

### 完整性检查

- [ ] 核心试验和支持性试验是否明确区分？
- [ ] 试验设计的关键要素是否完整？
- [ ] 主要终点的定义是否清晰？
- [ ] 统计方法是否详细描述？
- [ ] 主要结果是否包含效应量和置信区间？
- [ ] 监管机构的关注点是否记录？
- [ ] 特殊统计方法是否说明？

### 准确性检查

- [ ] 数据是否与原始文档一致？
- [ ] 计算是否正确？
- [ ] 结论是否得到数据支持？
- [ ] 引用是否准确？

### 清晰性检查

- [ ] 语言是否简洁明了？
- [ ] 术语使用是否一致？
- [ ] 结构是否逻辑清晰？
- [ ] 是否便于他人理解？

## 最佳实践

### 1. 保持客观

- 如实报告结果，不夸大或缩小
- 记录正面和负面的发现
- 包含不确定性和局限性

### 2. 注明来源

- 标注数据来源（页码、表格号）
- 区分申请人分析和监管分析
- 记录任何数据处理或转换

### 3. 关注监管视角

- 记录监管机构的关切
- 注意审稿人的评论
- 评估决策的依据

### 4. 更新维护

- 随着新信息出现更新总结
- 记录版本变更
- 保持文档的时效性
